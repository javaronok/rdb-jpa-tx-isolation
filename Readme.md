Проверяем работу уровня изоляции (SERIALAZIBLE) транзакции для СУБД RedDatabase. 
Для этого в одном потоке читаем все данные из таблицы STAT_DATA, считаем количество записей, ждём пока второй поток добавит строку в данные и считаем снова.
При уровне изоляции READ_COMMITED результаты первого потока должны измениться, при уровне изоляции SERIALAZIBLE результаты первогои второго запросы должны быть идентичны.

При уровне SERIALAZIBLE вставка новых данных завершается ошибкоц блокировки.
Видимо таблица блокируется поностью.

Но при уровне REPEATABLE_READ выполняются все условия изоляции SERIALAZIBLE (в том числе и фантомные записи)
